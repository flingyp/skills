# 需求技术方案 Skill

帮助 AI 生成结构化的技术方案文档，用于开发任务开始前的技术规划。

## 工作流程

1. **理解需求**：分析用户需求，明确当前结构和目标结构
2. **分析代码**：评估组件独立性、耦合度、外部调用影响
3. **制定方案**：设计实现步骤、文件变更清单
4. **评估风险**：识别潜在风险和应对措施
5. **生成文档**：输出到 `docs/{feature-name}-plan.md`

## 文档结构

详见 [references/template.md](references/template.md)，包含以下章节：

| 章节 | 内容 |
|------|------|
| 需求背景 | 当前结构 vs 目标结构对比 |
| 技术分析 | 组件独立性、可行性、影响评估（表格形式） |
| 实现方案 | 配置变更、文件变更清单 |
| 详细步骤 | 带代码示例的分步实现 |
| 风险评估 | 风险项、等级、应对措施 |
| 测试要点 | 功能测试清单 |
| 回滚方案 | Git 回滚文件清单 |

## 分析方法

### 组件独立性分析

使用表格形式分析：

```
| 组件 | Props | 状态管理 | API 调用 | 耦合度 |
|------|-------|----------|----------|--------|
| ComponentA | ... | ... | ... | 低/中/高 |
```

### 外部调用影响评估

```
| 调用位置 | 调用方式 | 影响评估 |
|----------|----------|----------|
| file:line | method() | 无影响/需修改 |
```

### 风险评估

```
| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| 描述 | 低/中/高 | 具体措施 |
```

## 输出规范

- **目录**：`docs/`
- **命名**：`{feature-name}-plan.md`
- **编码**：UTF-8
- **格式**：GitHub Flavored Markdown

## 使用示例

用户输入：
```
帮我规划一下将用户模块从主应用抽离的技术方案
```

AI 将：
1. 分析当前用户模块的代码结构
2. 评估组件依赖和外部调用
3. 生成完整的技术方案文档
