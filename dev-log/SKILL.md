# 开发日志 Skill

帮助 AI 生成简洁的开发日志文档，用于开发任务完成后的变更记录归档。

## 工作流程

1. **收集变更**：识别新建/修改/删除的文件
2. **整理细节**：记录关键技术决策和结构变更
3. **生成文档**：输出到 `logs/{feature-name}-{YYYY-MM-DD}.md`

## 文档结构

详见 [references/template.md](references/template.md)，包含以下章节：

| 章节 | 内容 |
|------|------|
| 需求概述 | 一句话描述需求目标 |
| 变更文件 | 新建/修改文件清单表格 |
| 结构变更 | 变更前/变更后对比（可选） |
| 技术细节 | 关键技术决策点 |
| 测试要点 | Checkbox 格式测试清单 |
| 开发时间 | 日期、开发人员 |

## 记录原则

- **简洁**：避免冗余，只记录关键信息
- **结构化**：使用表格和代码块
- **可追溯**：文件路径精确到具体位置

## 输出规范

- **目录**：`logs/`
- **命名**：`{feature-name}-{YYYY-MM-DD}.md`
- **日期格式**：YYYY-MM-DD（如 2026-02-27）
- **编码**：UTF-8
- **格式**：GitHub Flavored Markdown

## 使用示例

用户输入：
```
生成开发日志，记录刚才的背调体检 Tab 抽离变更
```

AI 将：
1. 分析本次会话中的代码变更
2. 整理变更文件和技术细节
3. 生成简洁的开发日志文档
