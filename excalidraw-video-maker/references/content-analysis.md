# 内容解析规则

将用户输入的主题或内容解析为结构化的视频素材。

## 解析步骤

### 1. 识别内容类型

| 类型 | 特征 | 适用图表 |
|------|------|----------|
| 技术讲解 | 概念定义、架构、数据流 | 流程图、架构图 |
| 业务流程 | 步骤、决策、角色 | 流程图、泳道图 |
| 教育培训 | 知识点、层级关系 | 思维导图、层级图 |
| 产品演示 | 功能模块、用户路径 | 流程图、对比图 |
| 时间线 | 事件序列、演进过程 | 时间线图 |

### 2. 提取核心元素

从内容中提取：

```
- 核心概念: 3-7 个关键名词
- 关系类型: 包含、依赖、顺序、对比、因果
- 层级结构: 顶层主题 → 子主题 → 细节
- 关键数据: 需要强调的数字、百分比
```

### 3. 拆分为场景

原则：
- 每个场景聚焦一个核心概念
- 单场景时长：5-15 秒
- 总场景数：3-8 个

拆分逻辑：

```
开场场景 (5-8s)
    │ 主题引入、问题提出
    ▼
主体场景 (每个 8-15s)
    │ 概念1 → 概念2 → 概念3 ...
    │ 按逻辑顺序或重要性排列
    ▼
结尾场景 (5-8s)
    │ 总结、行动建议
```

### 4. 确定叙事节奏

| 场景类型 | 建议时长 | 动画特点 |
|----------|----------|----------|
| 标题/开场 | 5-8s | 快速出现，简洁有力 |
| 核心概念 | 10-15s | 渐进展示，重点强调 |
| 流程步骤 | 8-12s | 按序绘制，箭头引导 |
| 对比分析 | 10-15s | 左右对比，突出差异 |
| 总结收尾 | 5-8s | 回顾要点，淡出结束 |

## 示例

**输入：**
> 解释 OAuth 2.0 授权流程

**解析输出：**

```yaml
content_type: 技术讲解
core_concepts:
  - 用户 (User)
  - 客户端 (Client)
  - 授权服务器 (Auth Server)
  - 资源服务器 (Resource Server)
  - Access Token
relationships:
  - 用户 → 授权 → 客户端
  - 客户端 → 请求 → 授权服务器
  - 授权服务器 → 返回 → Token
  - 客户端 → 携带 Token → 资源服务器
scenes:
  - id: intro
    type: 开场
    duration: 6s
    content: "OAuth 2.0 授权流程"
    narration: "今天我们来聊聊 OAuth 2.0，一种安全的授权协议。"
  - id: actors
    type: 概念
    duration: 10s
    content: "四个角色：用户、客户端、授权服务器、资源服务器"
    narration: "整个流程涉及四个角色：用户、客户端、授权服务器，以及资源服务器。"
  - id: flow
    type: 流程
    duration: 12s
    content: "授权码流程的四个步骤"
    narration: "接下来看看授权码流程，这是最常用的授权方式。第一步，用户点击授权；第二步，客户端获取授权码；第三步，用授权码换取令牌；第四步，使用令牌访问资源。"
  - id: summary
    type: 结尾
    duration: 5s
    content: "安全授权，无需共享密码"
    narration: "通过这种方式，用户密码永远不需要告诉第三方应用，安全又方便。"
```
