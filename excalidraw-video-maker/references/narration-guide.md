# 配音文本编写指南

为视频场景生成适合朗读的配音文本，输出为 JSON 格式。

## 文本编写原则

### 1. 口语化

配音文本应像日常对话，而非书面文章：

| 避免 | 推荐 |
|------|------|
| "该方法实现了..." | "这个方法能..." |
| "综上所述" | "总的来说" |
| "首先、其次、最后" | "第一步、然后、最后" |
| "进行配置" | "设置一下" |

### 2. 简洁明了

每句话控制在 15-20 字以内，避免长句：

```
差: OAuth 2.0 是一种授权框架，允许第三方应用在用户授权下访问其资源，而无需共享密码。

好: OAuth 2.0 让第三方应用能安全访问你的数据，而且不用告诉你密码。
```

### 3. 节奏感

- 短句为主，适当穿插长句
- 重要信息放在句首或句尾
- 用停顿（逗号）分隔信息块

### 4. 呼应视觉

配音内容与画面同步：

| 画面元素 | 配音配合 |
|----------|----------|
| 标题出现 | 先说主题名称 |
| 流程箭头 | 用"然后"、"接着"引导 |
| 对比两栏 | 用"左边"、"右边"指代 |
| 数据强调 | 放慢语速，加重语气 |

## 字数与时长对应

### 计算公式

```
建议字数 = 场景时长(秒) × 3
```

### 示例对照表

| 时长 | 建议字数 | 示例 |
|------|----------|------|
| 5s | 15 字 | "今天我们来聊聊 OAuth 2.0。" |
| 8s | 24 字 | "整个流程涉及四个角色，我们来逐一认识。" |
| 10s | 30 字 | "用户点击授权按钮，告诉服务器：我同意让这个应用访问我的数据。" |
| 12s | 36 字 | "第一步，用户发起请求；第二步，获取授权码；第三步，换取令牌；第四步，访问资源。" |
| 15s | 45 字 | "左边是传统方式，密码要告诉每个应用；右边是 OAuth 方式，密码只给授权服务器。明显右边更安全。" |

## JSON 输出格式

```json
{
  "videoTitle": "视频标题",
  "totalDuration": "45s",
  "scenes": [
    {
      "id": "scene-identifier",
      "order": 1,
      "duration": "6s",
      "startFrame": 0,
      "endFrame": 180,
      "narration": "这段是对应场景的配音文本。"
    }
  ],
  "fullNarration": "这是完整配音文本，将所有场景的 narration 按顺序拼接，便于一次性录音。"
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `videoTitle` | string | 视频标题 |
| `totalDuration` | string | 总时长（如 "45s"） |
| `scenes` | array | 场景数组 |
| `scenes[].id` | string | 场景唯一标识 |
| `scenes[].order` | number | 场景顺序（从 1 开始） |
| `scenes[].duration` | string | 该场景时长 |
| `scenes[].startFrame` | number | 起始帧（fps=30） |
| `scenes[].endFrame` | number | 结束帧 |
| `scenes[].narration` | string | 该场景配音文本 |
| `fullNarration` | string | 完整配音文本（所有场景拼接） |

## 场景类型配音模板

### 开场场景

```
模板: "[引入词] [主题]，[引导语]。"

示例:
- "今天我们来聊聊 REST API，看看它是怎么工作的。"
- "什么是微服务？让我们用一个例子来说明。"
```

### 概念讲解场景

```
模板: "[概念名称] 是 [简短定义]。[补充说明或例子]。"

示例:
- "客户端就是用户使用的应用，比如手机上的 App。"
- "授权服务器负责验证用户身份，并发放访问令牌。"
```

### 流程演示场景

```
模板: "第一步，[动作1]；第二步，[动作2]；第三步，[动作3]。"

示例:
- "第一步，用户点击授权；第二步，服务器返回授权码；第三步，客户端用授权码换取令牌。"
```

### 对比分析场景

```
模板: "左边是 [方案A]，[特点]；右边是 [方案B]，[特点]。"

示例:
- "左边是传统登录，密码要给第三方；右边是 OAuth 登录，密码只给自己人。"
```

### 结尾场景

```
模板: "[总结句]。[行动建议或结尾语]。"

示例:
- "现在你理解 OAuth 了吗？下次登录时，留意一下授权页面吧。"
- "安全授权，就是这么简单。感谢观看！"
```

## 质量检查清单

生成配音文本后，逐项检查：

- [ ] 每段文本字数符合时长要求
- [ ] 使用口语化表达，无书面语
- [ ] 句子简短，无明显长句
- [ ] 内容与画面元素对应
- [ ] 场景间过渡自然
- [ ] 完整文本可独立朗读
