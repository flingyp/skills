# Viral Tech Copywriter

## Quick Start

Generate viral copy by following this workflow:

1. **Gather tool/platform info**: Name, core features, GitHub stars, target users
2. **Choose formula** (see below): Pain-point formula OR Anxiety formula OR Goal formula
3. **Draft with style**: Apply [style-guide.md](references/style-guide.md) principles
4. **Validate against criteria**: Check all 6 key elements exist

**Output format**: 3-4 concise sentences, 150-200 characters

## Copywriting Formulas

### Formula A: 痛点型（直击痛点+对比传统）

**标准模板：**
```
Hook: 这个工具让你[最痛的具体场景]解放双手了
Compare: 相比传统的[旧方式]，它不仅[核心优势]
Features: [功能亮点1]、[功能亮点2]、[功能亮点3]
Closer: [流行语收尾]，[GitHub星数]星，[形容词]神器
```

**变体1（反问式）：**
```
Hook: 难道还要用手动改配置文件？[工具名]让你一键搞定
Compare: 再也不用[旧方式]那种让人头大的操作
Features: [功能1]自动搞定、[功能2]一键同步
Closer: 太爽了，[星数]星，[流行语]
```

**变体2（夸张承诺式）：**
```
Hook: 这个工具让你的[场景]效率提升10倍
Compare: 告别传统的[旧方式]繁琐流程
Features: [功能1]智能处理、[功能2]自动完成
Closer: [星数]星，[流行语]神器
```

**结构原则：**
- 开头：具体使用场景 + 解放/效率提升
- 中间：对比传统 + 2-3个功能（可变体表达）
- 结尾：流行语 + 星标 + 形容词

### Formula B: 焦虑型（制造紧迫+情绪共鸣）

**标准模板：**
```
Hook: 再不试试[工具名]你就落后了，[圈子]正在悄悄换赛道
Intro: 这个[工具名]框架不愧是拿下[星数]颗星的[流行语]
Features: [核心功能]几行代码搞定，[高级功能]直接调用
Compare: 以前[旧方式]让人头大要几百行，现在几十行搞定
Closer: 巧妙避开了[难点]，[情感共鸣]，真开发者的[流行语]
```

**变体1（数据冲击式）：**
```
Hook: 看到别人都在用[工具名]，你还没跟上速度？
Intro: [工具名]已经拿下[星数]颗星了，[流行语]
Features: [功能1]轻松搞定，[功能2]一键完成
Compare: 以前[旧方式]搞半天，现在几秒出结果
Closer: 别再落后了，[流行语]！
```

**变体2（圈子压力式）：**
```
Hook: 你的同事都在用[工具名]，就你还在[旧方式]
Intro: [星数]颗星的选择，[流行语]工具
Features: [功能1]、[功能2]、[功能3]全都有
Compare: 终于不用再[旧方式]让人抓狂了
Closer: [流行语]，跟上节奏吧！
```

**结构原则：**
- 开头：制造焦虑情绪（可选变体）
- 中间：功能密集 + 对比反差（可灵活组合）
- 结尾：情感共振 + 流行语（可强调紧迫感）

### Formula C: 目标型（明确目标+功能罗列）

**标准模板：**
```
Hook: [痛点场景]，这个开源工具让你直接[达成目标]
Intro: 一款专为[目标用户]打造的[工具类型]，[功能列表]
Mission: 目标很明确，让[任务]变得像[比喻]一样爽
Closer: [星数]星，[流行语+形容词]
```

**变体1（使命宣言式）：**
```
Hook: 还在为[痛点]烦恼？[工具名]来拯救你了
Intro: 专为[用户]打造的神器，[功能列表]
Mission: 让[任务]变得像吃饭喝水一样简单
Closer: [星数]星，[流行语]！
```

**变体2（直接切入式）：**
```
Hook: [工具名]，专治[痛点]
Intro: 支持一网打尽：[功能1]、[功能2]、[功能3]
Mission: 像[比喻]一样，爽！
Closer: [星数]星，[流行语][形容词]
```

**结构原则：**
- 开头：痛点 + 目标宣言（可选变体角度）
- 中间：功能密集罗列（可控制数量和表达方式）
- 结尾：宣言 + 流行语 + 数据（可强调使命）

### 混合策略（打破公式限制）

**允许跨公式组合：**
```
Formula A开头 + Formula B中间 + Formula C结尾
Formula B开头 + Formula A中间 + Formula C结尾
等灵活组合...
```

**组合原则：**
- 选择最强的Hook（痛点/焦虑/数据冲击）
- 中间功能表达选择最适合的方式（罗列/场景/对比）
- 结尾选择最合适的情感共鸣（流行语/形容词/数据）

**示例：**
```
Hook (Formula B): 再不试试就落后了，圈子正在换赛道
Middle (Formula A): 这个工具让你一键搞定，相比传统的手动操作，它支持[功能1]、[功能2]
Ending (Formula C): 目标很明确，让[任务]像吃饭一样简单，1.3万星，真香！
```

**使用原则：**

- 同一工具可尝试2-3种不同开场白
- 避免在一个文案中堆砌多个开场白变体

## Essential Elements

Every viral copy MUST include:

1. **痛点/钩子**：前3秒抓住注意力（解放双手、不会部署、落后了）
2. **流行语**：至少1-2个网络热词（纯牛马、真香、战神、福音）
3. **数据背书**：GitHub星标（6万多星、上万星、几千星）
4. **对比反差**：传统vs新工具、数量级对比（几百行→几十行）
5. **功能亮点**：2-4个核心功能，口语化表达
6. **口语化**：简洁短句，大白话节奏快

## Style Guidelines

详见 [style-guide.md](references/style-guide.md) 完整指南。

**Quick reference key principles:**

- **开篇钩子**：痛点、焦虑或夸张承诺
- **对比技巧**：传统vs新、数量对比、效果对比
- **功能表达**：罗列式、场景式、对比式
- **流行语库**：纯牛马、真香、战神、福音、战斗机
- **数据位置**：文案最后收尾，使用"星数+万/千"模糊化
- **节奏控制**：15-20字 → 30-40字 → 40-50字 → 20-30字

## Examples

### 输入要求
用户提供：
- 工具/项目名称
- 核心功能（2-4个）
- GitHub链接或星标数
- 目标用户群体
- 传统对比（可选）

### 生成模板

Formula A example:
```
这个工具让你[痛点场景]解放双手了。相比传统的[旧方案]，它不仅[功能1]，同时还支持[功能2]、[功能3]。[流行语]，[星数]星，[形容词]神器。
```

Formula B example:
```
再不试试[工具名]你就落后了，[圈子]正在悄悄换赛道。这个[工具名]不愧是拿下[星数]颗星的[流行语]。[功能1]几行搞定，[功能2]直接调用，以前[旧方式]让人头大要几百行，现在几十行搞定，巧妙避开了[难点]，[情感]，真香。
```

Formula C example:
```
[痛点场景]，这个开源工具让你直接[达成目标]。专为[用户]打造的[类型]，支持[功能1]、[功能2]、[功能3]、[功能4]。目标很明确，让[任务]变得像[比喻]一样爽。[星数]星，[流行语][形容词]。
```

## Usage Tips

1. **选择公式**：根据工具特点
   - 痛点明显 → Formula A
   - 追求差异化 → Formula B  
   - 目标导向强 → Formula C

2. **灵活适配**：
   - 少功能 → 不够3个，用2个
   - 无星标 → 用"获赞数"、"关注者"等数据
   - 多功能 → 用顿号密集罗列，控制在4个内

3. **避免模式化**：
   - 流行语要自然融入
   - 对比要真实有力
   - 句式可灵活调整

4. **数据表达**：
   - 5万+ → "6万多"、"5万多"
   - 1-5万 → "上万星"、"两万多星"
   - 1000-1万 → "几千星"、"几千颗星"
